# Security Policy

## Reporting a Vulnerability

If you find a security issue in this content — an unsafe command, a broken verification step, or a hardening recommendation that doesn't actually hold up — please open an [issue](https://github.com/Sanexxxx777/server-hardening-playbook/issues), or for sensitive reports contact **Aleksandr Shulgin** directly via [@Aleksandr_NFA](https://t.me/Aleksandr_NFA) rather than filing publicly.

## Scope

This repository is documentation and shell scripts for hardening Linux servers — there is no running service or application to compromise. The main risk is a recommendation that is wrong, outdated, or unsafe to apply without understanding it. Please report anything in that category.

## Disclosure

This is a personal open-source project with no fixed SLA, but reports are reviewed as soon as possible.
